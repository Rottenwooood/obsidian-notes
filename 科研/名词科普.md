- 消融
	- 有目的地移除或禁用模型中的某个组件、策略或参数设置，来观察其对最终结果的影响
	- 消融实验 = 移除某功能 → 观察性能变化 → 判断该功能的重要性
	- 验证某模块是否有效
	- 如在强化学习中原本学习率随时间衰减，测试学习率固定的情况，相对比
- 1. Rollout

- **定义**：在机器人学习和强化学习中，"rollout" 通常指模型（或策略）在环境中执行一个完整的轨迹或序列动作的过程，即从初始状态开始，通过连续决策生成一组观察-动作对，直到任务结束或终止。这类似于“ rollout of a policy ”，用于测试模型的实际性能、泛化能力和 emergent capabilities。
- **论文中的具体含义**：在RT-2论文中，rollout 特指模型的实际执行演示或模拟运行，特别是用于展示链式思考（chain-of-thought）推理的例子。例如，在Section 4.4中，论文提到“Rollouts of RT-2 with chain-of-thought reasoning are shown in Figure 7 and in Appendix I.” 这里，rollout 是模型在真实或模拟环境中响应指令（如“Pick the object that is different from all other objects”）生成的序列输出，包括计划步骤和动作token，然后转化为机器人行为。Rollout 用于定性评估模型的推理能力，如Figure 7中展示的从提示到预测的完整过程（e.g., “Plan: pick rxbar chocolate. Action: 1 128 129 125 131 125 128 127”）。
- **为什么重要**：Rollout 帮助验证模型是否能将web-scale知识转移到物理控制中，而非仅在训练数据上表现好。论文中约6000次评估轨迹（evaluation trajectories）本质上就是多次rollout的聚合结果。

2. Co-Fine-Tuning

- **定义**：Co-fine-tuning（也称联合微调或co-training with fine-tuning）是一种训练策略，指在fine-tuning（微调）特定任务数据（如机器人轨迹）的同时，继续包含并平衡预训练数据（如互联网规模的视觉-语言数据），以防止模型“灾难性遗忘”（catastrophic forgetting）原有知识。
- **论文中的具体含义**：在Section 3.2（Robot-Action Fine-tuning）中，论文提出这是RT-2训练的关键技术细节：“a key technical detail of the training recipe that improves robot performance is co-fine-tuning robotics data with the original web data instead of naïve finetuning on robot data only.” 具体过程：
    - 输入：机器人轨迹数据（图像 + 指令）与web数据（如VQA任务）混合。
    - 平衡：通过增加机器人数据集的采样权重，确保每个训练批次中机器人数据和web数据的比例均衡。
    - 益处：暴露模型于抽象视觉概念（从web数据）和低级机器人动作（从轨迹数据），提升泛化性能。消融实验（Section 4.3, Figure 6b）显示，co-fine-tuning 优于仅用机器人数据fine-tuning（~20-30% 性能提升），因为它保留了VLM的语义理解。
- **为什么重要**：机器人数据有限（~数百万轨迹），而web数据亿级；co-fine-tuning 让RT-2继承VLMs的泛化，同时学习动作输出，避免仅机器人训练导致的过拟合。

 3. 开放词汇（Open-Vocabulary）

- **定义**：Open-vocabulary 指模型能够处理或识别未在训练数据中明确见过的词汇或类别，而非局限于固定词汇表（closed-vocabulary）。在视觉-语言模型中，这意味着模型能进行开集（open-set）识别，如基于自然语言描述泛化到新对象，而无需重新训练。
- **论文中的具体含义**：在引言和Section 2（Related Work）中，论文多次提及VLMs的“open-vocabulary visual recognition”（e.g., 引用Radford et al., 2021的CLIP）。RT-2利用此能力扩展到机器人控制，例如“enable open-vocabulary visual recognition”（Section 1）。具体例子：
    - RT-2能解释未见于机器人训练数据的命令，如“placing an object onto a particular number or icon”（符号理解），或识别新对象（如“strawberry into the correct bowl”）。
    - 这源于VLMs的预训练（如PaLI-X、PaLM-E），允许模型泛化到“novel objects, semantically varied instructions”（Section 1）。
- **为什么重要**：机器人环境多样，固定词汇模型易失败；open-vocabulary 让RT-2实现emergent capabilities，如在未见场景中进行语义推理（Figure 2）。

 4. 全词汇（Full Vocabulary）

- **定义**：Full vocabulary 指模型在生成输出时可以使用完整的token词汇表，而非受限子集。这通常与输出约束相对，在多任务训练中用于区分不同模式（如语言任务 vs. 动作任务）。
- **论文中的具体含义**：在Section 3.2（Output Constraint）中，论文描述RT-2的输出机制：“to ensure that RT-2 outputs valid action tokens during decoding, we constrain its output vocabulary via only sampling valid action tokens when the model is prompted with a robot-action task, whereas the model is still allowed to output the full range of natural language tokens on standard vision-language tasks.”
    - “Full vocabulary” 特指在VQA（视觉-语言任务）中允许全范围自然语言token（e.g., 数亿token的PaLM-E词汇）。
    - 与之对比，机器人任务时限制为动作token（256个），确保输出有效动作。
- **为什么重要**：这允许RT-2在co-fine-tuning中共享权重：语言任务用全词汇保持VLM能力，动作任务用约束避免无效输出，提升实用性。

 5. 输出的动作具体如何表示

- **定义**：在RT-2中，机器人动作被表示为离散化的文本token序列，与自然语言统一，便于端到端训练。这是一种“将动作作为另一种语言”（actions as another language）的设计（Figure 1）。
- **论文中的具体表示**（Section 3.2）：
    - **动作空间**：6DoF（自由度）末端执行器位移（Δposition x/y/z）+ 旋转（Δrotation x/y/z）+ 夹持器伸展（gripper extension）+ 离散终止命令（terminate episode）。
    - **离散化**：连续维度（除终止）均匀离散为256 bin（0-255整数）。终止用特殊token（如“terminate”）。
    - **字符串格式**：将8个整数（7动作 + 1终止）串联，用空格分隔，后缀“terminate”。示例：“1 128 91 241 5 101 127 terminate”。
        - 解释：第一个“1”可能表示终止bin；后续是Δpos_x (128)、Δpos_y (91)、Δpos_z (241)、Δrot_x (5)、Δrot_y (101)、Δrot_z (127)；gripper_extension 隐含在序列中。
    - **Token映射**：
        - PaLI-X：直接用数字token（整数0-1000有唯一token）。
        - PaLM-E：覆盖256个最低频token，实现symbol tuning。
    - **训练输入/输出**：输入格式：“Q: What should the robot do to <task>? A: <img>”，输出：上述动作字符串。推理时，反token化（de-tokenize）成实际动作（如ΔT = [0.1, -0.2, 0], ΔR = [10°, 25°, -7°]，Figure 1）。</task>
    - **示例**（Figure 7 rollout）：“Action: 1 128 124 136 121 158 111 255.” 这对应一个具体位移/旋转序列，用于pick对象。
- **为什么重要**：统一表示让VLMs直接输出动作，无需额外架构；支持闭环控制（closed-loop），频率1-5Hz。
